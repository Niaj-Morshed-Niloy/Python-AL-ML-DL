Topic 01

Why RNNs for Natural Language Processing?

Language is inherently sequential: a word's meaning depends on the words around it. A feedforward network treats each input independently and has no memory of order — RNNs add a hidden state that carries context from one token to the next.

Feedforward vs Recurrent — the key difference
Feedforward network
Recurrent network
The recurrent edge (animated) is the whole idea: the hidden state feeds back so the network "remembers" earlier tokens.

Common NLP tasks with RNNs

  • Language modeling — predict the next word: P(wt | w1, …, wt−1)
  • Text classification — sentiment analysis, spam detection, intent recognition
  • Sequence labeling — part-of-speech tagging, named entity recognition (NER)
  • Machine translation — encoder RNN reads source; decoder RNN generates target (seq2seq)
Example — why order matters
A bag-of-words or feedforward model sees the same tokens and cannot tell these apart; an RNN reads them in order and keeps context:
  • "dog bites man" vs "man bites dog" — identical words, opposite meaning.
  • Context resolves ambiguity: "I deposited cash at the bank" vs "we sat on the river bank".
  • Long-range agreement: "The keys that were on the table are mine" (not "is") — the verb depends on a far-away word.
Key idea: An RNN reuses the same weights at every time step and updates a hidden state that summarizes everything seen so far — ideal for variable-length text.
Topic 02

Inside the RNN Cell

At each time step t the cell takes the current input xt and the previous hidden state ht−1, mixes them with shared weight matrices, applies tanh, and emits a new hidden state ht (and optionally an output ŷt).

Single RNN cell — inputs, output, and the recurrent feedback
input / Wxh hidden state h output / Why

Core equations (vanilla RNN)

hₜ = tanh( W_xh xₜ + W_hh hₜ₋₁ + bₕ )
ŷₜ = softmax( W_hy hₜ + bₙ )
Example — one forward step by hand
A tiny RNN with a 2-dim hidden state, scalar input, zero biases, and h₀ = [0, 0]: 
W_xh = [0.5, -0.3]      W_hh = [[0.1, 0], [0, 0.1]]      x₁ = 1

h₁ = tanh(W_xh·x₁ + W_hh·h₀) = tanh([0.5, -0.3]) ≈ [0.46, -0.29]
# next step mixes the NEW input x₂ with W_hh·h₁ — that carry-over is the "memory"
Intuition: the hidden state is like a running mental summary of a sentence — each new word updates the summary instead of re-reading everything from scratch.
Weight Sharing
W_xh, W_hh, W_hy are the same at every step — parameter count is independent of sequence length.
Hidden State = Memory
ht is a fixed-size vector that compresses everything the model has read up to step t.
Checkpoint: Which matrix is responsible for carrying information across time? (Answer: W_hh.)
Topic 03

Unrolling Through Time

To process a sentence, we "unroll" the single cell into one copy per token. The copies share weights and are chained by the hidden state: h₀ → h₁ → h₂ → …. Press Play to build the graph step by step.

Unrolled RNN over the sequence "The cat sat ."
Each column is the same cell at a different time step. Horizontal arrows carry the hidden state forward; the weights are identical across all columns.

RNN input/output layouts in NLP

Many-to-one (sentiment)
flowchart LR
  x1["x₁"] --> h1((h₁))
  x2["x₂"] --> h2((h₂))
  x3["x₃"] --> h3((h₃))
  h1 --> h2 --> h3 --> y["ŷ"]
One-to-many (captioning)
flowchart LR
  x["x"] --> h1((h₁)) --> h2((h₂)) --> h3((h₃))
  h1 --> y1["ŷ₁"]
  h2 --> y2["ŷ₂"]
  h3 --> y3["ŷ₃"]
Many-to-many synced (POS / NER)
flowchart LR
  x1["x₁"] --> h1((h₁)) --> y1["ŷ₁"]
  x2["x₂"] --> h2((h₂)) --> y2["ŷ₂"]
  x3["x₃"] --> h3((h₃)) --> y3["ŷ₃"]
  h1 --> h2 --> h3
Seq2seq (translation)
flowchart LR
  x1["x₁"] --> e1((h₁)) --> e2((h₂)) --> c{{context}}
  c --> d1((s₁)) --> y1["ŷ₁"]
  d1 --> d2((s₂)) --> y2["ŷ₂"]
Example — each layout in the wild
  • Many-to-one: a movie review → one sentiment label (positive / negative).
  • One-to-many: one image → a caption "a dog on a couch".
  • Many-to-many (synced): POS tagging → "The/DET cat/NOUN sat/VERB".
  • Seq2seq: translation → "I am happy" → "Je suis heureux" (output length differs from input).
Same cell, different wiring: all four layouts use one recurrent cell; only where we read inputs and emit outputs changes.
Topic 04

Forward Propagation of RNN

Forward propagation sweeps left to right: starting from h₀ (usually zeros), each step computes a hidden state from the previous one and the current token, then an output. Press Play to watch the signal flow.

Forward pass — information flows forward in time
currently activating hidden state output
# for t = 1 … T
hₜ = tanh( W_xh xₜ + W_hh hₜ₋₁ + bₕ )
ŷₜ = softmax( W_hy hₜ + bₙ )
Example — predict the next word
Vocabulary {cat, sat, mat}. After reading "the cat", the output layer produces logits, softmax turns them into probabilities, and cross-entropy scores the prediction: 
logits([cat, sat, mat]) = [1.0, 2.0, 0.1]
softmax              ≈ [0.24, 0.66, 0.10]      → predicts "sat" ✓
loss = -ln(0.66)     ≈ 0.42                  # low loss: confident & correct
# if "sat" had only 0.24 prob:  -ln(0.24) ≈ 1.43  (much higher loss)
The loss is large when the correct word gets a low probability — and that error is exactly what backward propagation reduces.

NumPy implementation

forward.py 
import numpy as np

def rnn_forward(X, W_xh, W_hh, W_hy, b_h, b_y, h0=None):
    T, _ = X.shape
    H = W_hh.shape[0]
    h = np.zeros((T + 1, H))
    if h0 is not None: h[0] = h0
    y = np.zeros((T, W_hy.shape[0]))

    for t in range(T):
        h[t + 1] = np.tanh(X[t] @ W_xh + h[t] @ W_hh + b_h)
        y[t] = h[t + 1] @ W_hy + b_y
    return y, h   # keep all h for backprop
Tip: Store every ht during the forward pass — backward propagation needs them to compute gradients.
Topic 05

Backward Propagation Through Time (BPTT)

Training reverses the flow: starting from the loss, gradients travel right to left through every time step, accumulating updates for the shared weights. Press Play to watch gradients flow backward.

Backward pass — gradients flow backward in time
gradient signal hidden state

Vanishing / exploding gradients

The gradient flowing back to early steps is repeatedly multiplied by W_hh (and tanh slopes). If the effective factor is < 1 it vanishes; if > 1 it explodes. Drag the slider to see it.

Gradient magnitude reaching step (t − k)
per-step factor γ: 0.60
Gradient at distance k behaves like γk. This is why vanilla RNNs struggle with long-range dependencies — motivating LSTM and GRU.
Example — why long-range memory fails
With a per-step factor γ, the gradient reaching a token 20 steps back scales like γ20: 
γ = 0.5  →  0.5^20 ≈ 0.000001   # vanishes: early tokens barely learn
γ = 1.2  →  1.2^20 ≈ 38         # explodes: huge, unstable updates
Real consequence: "I grew up in France … [40 words] … so I speak fluent ___." The cue (France → French) is too far back for a vanilla RNN to retain — the core motivation for LSTM and GRU.

NumPy gradient sketch

bptt.py 
# walk t = T-1 down to 0; dh carries gradient back in time
dW_xh = dW_hh = dW_hy = db_h = db_y = 0
dh_next = np.zeros(H)

for t in reversed(range(T)):
    dy = y_pred[t] - y_true[t]            # softmax + cross-entropy
    dW_hy += np.outer(h[t+1], dy)
    db_y  += dy
    dh = dy @ W_hy.T + dh_next            # from output + future step
    dz = (1 - h[t+1]**2) * dh        # tanh derivative
    dW_xh += np.outer(X[t], dz)
    dW_hh += np.outer(h[t], dz)
    db_h  += dz
    dh_next = dz @ W_hh.T                 # pass gradient to t-1
Why sum? Because the same W_hh is used at every step, its gradient is the sum of contributions from all time steps.
Topic 06

RNN in PyTorch

In practice the forward/backward math is handled for you. Here is a complete many-to-one sentiment classifier: embed tokens, run an RNN, and classify from the final hidden state.

model.py 
import torch
import torch.nn as nn

class RNNClassifier(nn.Module):
    def __init__(self, vocab, emb=64, hid=128, classes=2):
        super().__init__()
        self.embed = nn.Embedding(vocab, emb)
        self.rnn   = nn.RNN(emb, hid, batch_first=True)
        self.fc    = nn.Linear(hid, classes)

    def forward(self, x):
        e = self.embed(x)              # (B, T, emb)
        out, h_n = self.rnn(e)         # out: (B, T, hid)
        return self.fc(h_n[-1])         # last hidden -> logits

model = RNNClassifier(vocab=5000)
x = torch.randint(0, 5000, (8, 20))  # batch 8, seq len 20
logits = model(x)                      # (8, 2)

loss = nn.CrossEntropyLoss()(logits, torch.zeros(8, dtype=torch.long))
loss.backward()                        # autograd runs BPTT for you
Example — follow the tensor shapes
For a batch of 8 sentences, each padded to seq_len = 20, with a 5000-word vocab (emb = 64, hid = 128): 
x          : (8, 20)          # token IDs
embed(x)   : (8, 20, 64)      # each token → 64-dim vector
rnn output : (8, 20, 128)     # hidden state at every step
h_n[-1]    : (8, 128)         # final hidden state per sentence
fc(...)    : (8, 2)           # 2-class logits per sentence
Shape mismatches are the #1 beginner bug — tracing each line above is what makes nn.RNN finally click.
Embedding
Maps token IDs to dense vectors — the xt fed into the RNN.
nn.RNN
Runs the forward recurrence; returns all outputs and the final hidden state h_n.
loss.backward()
Autograd performs BPTT automatically — the same gradients you coded by hand.
Swap-in upgrade: change nn.RNN to nn.LSTM or nn.GRU to fight vanishing gradients — the rest of the code stays the same.

Key takeaways

  • RNNs model sequential text via a hidden state updated at every token with shared weights.
  • Forward propagation unrolls the cell and computes ht, ŷt left to right.
  • BPTT sends gradients right to left, summing contributions for the shared weights.
  • Vanishing/exploding gradients limit long-range memory — LSTM/GRU address this.
Example — when does order matter?
  • Bag-of-Words: ignores order, so "not good" ≈ "good". ✗ order, ✗ memory.
  • Feedforward net: needs fixed-size input and is still order-agnostic. ✗ variable length.
  • RNN: ✓ order-aware, ✓ any length, ✓ short-range memory — long-range is the weak spot (→ LSTM/GRU, then Transformers).
